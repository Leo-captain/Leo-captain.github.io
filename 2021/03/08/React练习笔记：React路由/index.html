一、SPA的概念与理解：

1、单页web应用，single page web application，SPA

2、整个应用只有一个完整的页面

3、点击页面中的链接不会刷新页面，只会做页面的局部更新

4、数据都需要通过ajax获取，并在前端异步展现

三、前端路由的工作原理

在BOM浏览器对象上有一个history对象，专门用于管理浏览器的路径、历史记录等，使用原生API进行操作可能较为繁琐，因此可以使用一个库history.js对其进行操作，封装之后的history对象会拥有更多便捷的API。

如上面的代码所示，创建一个封装后的history对象后（可以使用createBrowserHistory()和createHashHistory()两种不同的方法创建），可以通过push方法向浏览器的历史记录当中压入一条新的内容，并且push方法最后返回一个false，这就可以让a标签原本的跳转功能失效，仅仅改变浏览器的地址，却不会跳转。

而history.listen这个方法开启之后则可以随时监听浏览器中地址的变化，通过监听到的地址变化，那么就可以将地址中对应的组件挂载到页面上，从而实现一个完整的React路由的功能。

八、Switch的使用

注册路由时，如果同一个路由对应了多个组件，那么匹配到这个路由的情况下就会把所有匹配到的组件挂载到页面上；此外，不管路由是否已经被匹配到了，React都会把所有的注册路由都遍历一遍，这样非常浪费时间。

为了解决这个问题，可以在注册路由的外侧加一组Switch标签，这样一旦匹配到第一个对应的路由，就不会再向下匹配了：
